Задание 1. Анализ HTTP-запросов. Часть 1

Откройте сайт Wikipedia: https://en.wikipedia.org/wiki/HTTP

Прочитайте статью и обратите внимание на разделы, посвященные HTTP-запросам.

Откройте инструменты разработчика в вашем браузере (обычно это можно сделать, нажав F12 или правой кнопкой мыши выбрав "Inspect").

Перейдите на вкладку "Network" (Сеть).

Обновите страницу Wikipedia.

Найдите первый запрос к странице, он будет назван как HTTP и изучите следующие аспекты:

URL запроса

URL запроса: https://en.wikipedia.org/wiki/HTTP

Метод запроса. Почему именно данный метод используется для загрузки страницы?

Метод запроса: GET. Он используется потому что браузер запрашивает страницу для чтения. GET это стандартный метод HTTP для запроса ресурса без изменения его на сервере.

Статус ответа. Что означает статус ответа?

Статус ответа: 200 OK. Это значит, что все прошло успешно. 200 OK - стандартный код успеха, сервер вернул страницу без ошибок.

Заголовки запроса и ответа. Какие заголовки присутствуют и какую информацию они содержат?

Заголовки запроса включают параметры, которые браузер отправляют серверу. В этом случае присутствуют заголовки: :authority (означает домен и содержит en.wikipedia.org), user-agent (браузер и устройство, содержит Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/144.0.0.0 Safari/537.36), accept (показывает какие типы данных браузер готов принять, содержит text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.7), accept-language (предпочтительный язык, содержит ru-RU,ru;q=0.9,en-US;q=0.8,en;q=0.7)

Заголовки ответа: content-type (означает, что это HTML-страница, содержит text/html; charset=UTF-8), date (содержит дату и время сервера), server (содержит информацию о сервере), content-length (размер тела ответа), cache-control (содержит информацию о том, как кешировать страницу), last-modified (данные для кеширования), strict-transport-security (принудительное HTTPS)

Есть ли тело запроса или ответа? Если да, то что в нем содержится?

Для URL https://en.wikipedia.org/wiki/HTTP при загрузке страницы используется метод GET, поэтому тело запроса отсутствует - браузер просто запрашивает страницу у сервера. В теле ответа сервер возвращает HTML-код статьи Wikipedia “HTTP”, который содержит структуру страницы, текст статьи, ссылки, таблицы и другие элементы. Именно это содержимое браузер затем отображает пользователю как готовую веб-страницу.

Какие еще запросы были отправлены при загрузке страницы и почему?

При загрузке страницы браузер также загружает CSS файлы для оформления статьи JavaScript для работы интерфейса Wikipedia, а также изображения и иконки, которые используются на странице. Эти запросы нужны потому, что сама HTML-страница содержит только структуру и текст, а внешний вид, стили и дополнительные элементы загружаются отдельно. Поэтому в Network видно не один запрос, а много - каждый отвечает за свою часть отображения страницы.
