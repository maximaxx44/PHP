# StudyTasks

## Описание проекта

**StudyTasks** — это веб-приложение для управления учебными заданиями. Проект разработан как лабораторная работа по созданию серверного веб-приложения средней сложности.

Приложение позволяет пользователям регистрироваться, входить в систему, создавать учебные задания, редактировать их, искать по критериям и управлять личным списком задач. Также в системе есть публичный раздел с материалами и административная панель для управления пользователями, категориями и заданиями.

Проект реализован на **PHP** с использованием **MySQL** и запускается локально через **XAMPP**.

## Инструкции по запуску проекта

### 1. Установка необходимых программ

Для запуска проекта необходимо установить:

* XAMPP;
* браузер;
* Git, если проект будет загружаться в репозиторий.

В XAMPP используются:

* Apache;
* MySQL;
* phpMyAdmin.

---

### 2. Размещение проекта

Скопируйте папку проекта `studytasks` в директорию:

```text
C:\xampp\htdocs\
```

```text
C:\xampp\htdocs\studytasks
```

Откройте **XAMPP Control Panel** и запустите:

```text
Apache
MySQL
```

Оба сервиса должны быть активны.

```text
http://localhost/phpmyadmin
```


3. Нажмите **Вперёд / Go**.

После импорта появится база данных:

```text

```

Откройте проект в браузере:

```text

```

## Описание лабораторной работы

Цель лабораторной работы — разработать веб-приложение средней сложности с серверной логикой, базой данных, регистрацией и авторизацией пользователей, защищёнными разделами, формами взаимодействия и административными функциями.

В рамках проекта реализовано приложение **StudyTasks**, предназначенное для управления учебными заданиями.

Основные задачи работы:

* реализовать регистрацию и вход пользователей;
* обеспечить безопасное хранение паролей;
* создать публичный раздел сайта;
* реализовать защищённый личный кабинет;
* создать форму добавления ресурса с валидацией;
* реализовать форму поиска;
* добавить роль администратора;
* реализовать управление данными в базе;
* подготовить проект к размещению на GitHub.

## Функциональные возможности

### Общедоступная часть

Доступна всем пользователям без авторизации.

Включает:

* главную страницу;
* публичные материалы;
* базу знаний;
* публичные задания;
* поиск по открытым заданиям.

Материалы и публичные задания загружаются из базы данных.

---

### Регистрация и вход

Пользователь может:

* создать аккаунт;
* войти в систему;
* выйти из системы;
* восстановить пароль.

Пароль сохраняется в базе данных только в виде хэша.

Для хэширования используется функция:

```php
password_hash($password, PASSWORD_DEFAULT);
```

Для проверки пароля используется:

```php
password_verify($password, $user['password_hash']);
```

---

### Личный кабинет пользователя

После входа пользователь получает доступ к личному кабинету.

В личном кабинете можно:

* просматривать свои задания;
* создавать новые задания;
* редактировать задания;
* удалять задания;
* видеть статистику по статусам;
* искать задания по критериям.

---

### Создание задания

Форма создания задания содержит несколько полей разных типов:

* название задания;
* описание;
* категория;
* приоритет;
* статус;
* дата сдачи;
* признак публичности.

В форме используется клиентская и серверная проверка данных.

---

### Поиск заданий

Поиск позволяет находить задания по следующим критериям:

* текст запроса;
* категория;
* статус;
* приоритет.

Если пользователь не авторизован, он видит только публичные задания. Если пользователь вошёл в систему, он видит публичные задания и свои личные задания.

---

### Административная панель

Для пользователей с ролью администратора доступна отдельная админ-панель.

Администратор может:

* просматривать всех пользователей;
* создавать нового администратора;
* редактировать пользователей;
* удалять пользователей;
* просматривать все задания;
* редактировать любые задания;
* удалять задания;
* управлять категориями.

---

## Сценарии взаимодействия пользователей с приложением

### Сценарий 1. Гость просматривает публичную часть

1. Пользователь открывает главную страницу.
2. Видит публичные материалы и открытые задания.
3. Может открыть материал из базы знаний.
4. Может воспользоваться поиском по публичным заданиям.

---

### Сценарий 2. Регистрация пользователя

1. Пользователь открывает страницу регистрации.
2. Вводит имя, email и пароль.
3. Система проверяет данные.
4. Пароль хэшируется.
5. Пользователь сохраняется в таблицу `users`.
6. После регистрации пользователь может войти в систему.

---

### Сценарий 3. Создание учебного задания

1. Пользователь входит в аккаунт.
2. Открывает страницу создания задания.
3. Заполняет форму.
4. Система проверяет данные на клиенте и сервере.
5. Задание сохраняется в таблицу `tasks`.
6. Пользователь возвращается в личный кабинет.

---

### Сценарий 4. Поиск задания

1. Пользователь открывает страницу поиска.
2. Вводит текст или выбирает фильтры.
3. PHP получает параметры через GET-запрос.
4. Выполняется SQL-запрос к таблице `tasks`.
5. Результаты отображаются на странице.

---

### Сценарий 5. Работа администратора

1. Администратор входит в систему.
2. Открывает админ-панель.
3. Управляет пользователями, заданиями и категориями.
4. Может создавать новых администраторов.
5. Может редактировать и удалять записи из базы данных.

### Пример подключения к базе данных через PDO

```php
$pdo = new PDO($dsn, $config['db_user'], $config['db_pass'], [
    PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
]);
```

Этот код создаёт подключение к базе данных MySQL и включает режим обработки ошибок через исключения.

### Пример безопасного SQL-запроса

```php
$stmt = db()->prepare('SELECT * FROM users WHERE email = ?');
$stmt->execute([$email]);
$user = $stmt->fetch();
```

Здесь используется подготовленный запрос. Данные пользователя не вставляются в SQL напрямую, поэтому снижается риск SQL-инъекций.

## Безопасность

В проекте реализованы следующие меры безопасности:

* пароли хранятся в виде хэша;
* используется `password_hash()` и `password_verify()`;
* доступ к закрытым страницам проверяется через сессию;
* административные страницы доступны только пользователям с ролью `admin`;
* данные из форм проверяются на стороне клиента и сервера;
* для SQL-запросов используется PDO и подготовленные выражения;
* вывод данных экранируется через `htmlspecialchars()`;
* токены восстановления пароля имеют срок действия и помечаются использованными после смены пароля.

## Восстановление пароля через SMTP

В проекте реализовано восстановление пароля через токен.

Процесс работы:

1. Пользователь вводит email.
2. Система ищет пользователя в базе данных.
3. Создаётся уникальный токен.
4. Токен сохраняется в таблицу `password_resets`.
5. Формируется ссылка для сброса пароля.
6. Ссылка отправляется пользователю на email через SMTP.
7. Пользователь открывает ссылку.
8. Вводит новый пароль два раза.
9. Если пароли совпадают, пароль обновляется.
10. Токен помечается как использованный.

SMTP-настройки находятся в файле:

```text
config/config.php
```

Пример настроек:

```php
'smtp_enabled' => true,
'smtp_host' => 'smtp.gmail.com',
'smtp_port' => 587,
'smtp_username' => 'example@gmail.com',
'smtp_password' => 'app_password',
'smtp_encryption' => 'tls',
'smtp_from_email' => 'example@gmail.com',
'smtp_from_name' => 'StudyTasks',
```

Перед загрузкой проекта на GitHub реальный пароль приложения нужно удалить из конфигурации.

## Ответы на контрольные вопросы

### 1. Что такое веб-приложение?

Веб-приложение — это программа, которая работает через браузер и взаимодействует с сервером. Пользователь отправляет запросы через интерфейс сайта, а сервер обрабатывает их и возвращает результат.

---

### 2. Что такое серверная логика?

Серверная логика — это код, который выполняется на сервере. В данном проекте серверная логика написана на PHP. Она отвечает за регистрацию, вход, работу с базой данных, проверку прав доступа и обработку форм.

---

### 3. Что такое база данных?

База данных — это организованное хранилище информации. В проекте используется MySQL, где хранятся пользователи, задания, категории, материалы и токены восстановления пароля.

---

### 4. Что такое аутентификация?

Аутентификация — это проверка личности пользователя. В проекте пользователь вводит email и пароль, после чего система проверяет данные и создаёт сессию.

---

### 5. Почему пароль нельзя хранить обычным текстом?

Если хранить пароль обычным текстом, при утечке базы данных злоумышленник сразу увидит все пароли. Поэтому пароль хранится в виде хэша.

---

### 6. Что такое хэширование пароля?

Хэширование — это преобразование пароля в специальную строку, из которой нельзя просто восстановить исходный пароль. В PHP для этого используется `password_hash()`.

---

### 7. Что такое сессия?

Сессия — это механизм хранения данных о пользователе между запросами. После входа в систему в сессии сохраняется ID пользователя, и сайт понимает, кто сейчас авторизован.

---

### 8. Что такое CRUD?

CRUD — это четыре основные операции с данными:

* Create — создание;
* Read — чтение;
* Update — обновление;
* Delete — удаление.

В проекте CRUD реализован для учебных заданий.

---

### 9. Что такое PDO?

PDO — это инструмент PHP для подключения к базе данных. В проекте PDO используется для работы с MySQL и выполнения безопасных подготовленных SQL-запросов.

---

### 10. Что такое SQL-инъекция?

SQL-инъекция — это атака, при которой злоумышленник пытается вставить вредный SQL-код через поля формы. Для защиты используются подготовленные запросы PDO.

---

### 11. Зачем нужна роль администратора?

Роль администратора нужна для разделения прав доступа. Обычный пользователь управляет только своими заданиями, а администратор может управлять пользователями, категориями и всеми заданиями.

---

### 12. Как работает восстановление пароля?

При запросе восстановления создаётся токен, который сохраняется в базе данных. Пользователь получает ссылку с токеном, открывает её и задаёт новый пароль. После успешной смены пароля токен становится недействительным.
