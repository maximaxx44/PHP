Лабораторная работа №6. Обработка и валидация форм

Цель работы

Освоить основные принципы работы с HTML-формами в PHP, включая отправку данных на сервер и их обработку, включая валидацию данных.

Условие

Студенты должны выбрать тему проекта для лабораторной работы, которая будет развиваться на протяжении курса.

Например:

Управление рецептами (добавление, редактирование, удаление рецептов);

Дневник настроения;

Трекер привычек;

Мини-система “тайных признаний”;

Каталог “странных фактов”;

Менеджер личных “анти-целей”;

Шаг 1. Определение модели данных

Для выбранной темы определите, какие данные будут храниться. Например, для “Управления рецептами” это может быть:

title (название рецепта);

ingredients (список ингредиентов);

instructions (пошаговое описание);

category (категория блюда);

prep_time (время приготовления);

difficulty (сложность).

created_at (дата создания);

updated_at (дата последнего обновления).

author (автор рецепта).

Примечание: при определение модели данных обязательно должно быть:

минимум 6 полей;

хотя бы 1 поле с типом string (текст);

хотя бы 1 поле с типом date (дата);

хотя бы 1 поле с типом enum (ограниченный набор значений) (checkbox);

хотя бы 1 поле с типом text (длинный текст).

Шаг 2. Создание HTML-формы

Разработайте HTML-форму для создания новой записи (например, нового рецепта). Форма должна содержать все необходимые поля, соответствующие модели данных.

Форма должна использовать метод POST и отправлять данные на сервер для обработки.

Добавьте базовую валидацию на стороне клиента (например, с помощью атрибутов required, minlength, maxlength и т.д.) для улучшения пользовательского опыта.

Убедитесь, что форма корректно отображается и работает в браузере.

Шаг 3. Обработка данных на сервере

Создайте PHP-скрипт для обработки данных, отправленных из формы. Этот скрипт должен:

принимать данные из $_POST;

выполнять базовую валидацию данных (например, проверять, что обязательные поля заполнены, что дата имеет правильный формат и т.д.);

сохранять данные в файл (например, data.txt);

возвращать пользователю сообщение об успешной отправке или об ошибках валидации.

Убедитесь, что данные сохраняются в читаемом формате (например, JSON или CSV) для удобства последующего использования.

Шаг 4. Вывод данных

Создайте PHP-скрипт для чтения данных из файла и отображения их в виде HTML-таблицы.

Таблица должна отображать все записи, сохранённые в файле, и быть отформатирована для удобства чтения (например, с помощью CSS).

Добавьте возможность сортировки данных по различным полям (например, по дате создания, по категории и т.д.) для улучшения пользовательского опыта.

Ответы на контрольные вопросы:

Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

Основными методами отправки данных из HTML-формы на сервер являются GET и POST.
Метод GET передаёт данные через адресную строку и чаще используется для поиска, фильтрации и сортировки. 
Метод POST передаёт данные в теле запроса и подходит для отправки форм, добавления записей и других операций, связанных с изменением данных. 
В данной лабораторной работе используется метод POST, так как форма предназначена для сохранения новой записи.

Какие глобальные переменные используются для доступа к данным формы в PHP?

В PHP для доступа к данным формы используются суперглобальные массивы $_GET, $_POST и $_REQUEST. 
Массив $_GET применяется при передаче данных методом GET, а $_POST — при отправке формы методом POST. 
Массив $_REQUEST может содержать данные сразу из нескольких источников, но на практике чаще рекомендуется использовать $_GET и $_POST, так как это делает код более понятным и предсказуемым. 
В данной работе данные формы обрабатываются через $_POST.

Как обеспечить безопасность при обработке данных из формы (например, защититься от XSS)?

Безопасность при обработке формы обеспечивается с помощью валидации, очистки и безопасного вывода пользовательских данных. 
Перед сохранением следует проверять корректность введённой информации, а также очищать её с помощью функций вроде trim() и strip_tags(). 
Для защиты от XSS при выводе данных на страницу необходимо использовать htmlspecialchars(), чтобы введённый пользователем текст не выполнялся как HTML или JavaScript-код. 
Таким образом, сочетание серверной валидации и безопасного вывода позволяет значительно повысить надёжность приложения.

Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

Данные из формы на сервер чаще всего отправляются методами GET и POST. Метод GET передаёт данные через адресную строку, поэтому его удобно использовать для поиска, фильтрации или сортировки. Метод POST передаёт данные в теле запроса, поэтому он лучше подходит для добавления записей, регистрации, авторизации и других действий, где данные не должны отображаться в ссылке.

Какие глобальные переменные используются для доступа к данным формы в PHP?

В PHP для получения данных из формы используются суперглобальные массивы $_GET и $_POST. Если форма отправлена методом GET, данные будут доступны в массиве $_GET, а если методом POST — в массиве $_POST. Также существует $_REQUEST, но лучше использовать конкретно $_GET или $_POST, чтобы код был понятнее и безопаснее.

Как обеспечить безопасность при обработке данных из формы?

При обработке формы нельзя доверять данным, которые вводит пользователь, поэтому их нужно проверять и очищать. Для защиты от XSS при выводе данных на страницу используется функция htmlspecialchars(), которая превращает HTML-теги в обычный текст. Также нужно проверять, что обязательные поля заполнены, числа действительно являются числами, а длина строк не превышает допустимые значения.
